# Zenn 技術ブログ執筆ガイド

Zenn向けの日本語技術ブログ記事を、調査から公開準備まで一貫して執筆するためのワークフロー。

---

## ワークフロー

### Phase 1: 調査

ユーザーのテーマに関する情報を収集・整理する。

1. **公式ドキュメント調査**: WebFetch / WebSearch で公式ドキュメント、GitHub リポジトリを調査
2. **情報整理**: 主要機能、セットアップ手順、制約事項を構造化してメモ
3. **ユーザー確認**: 調査結果をユーザーに共有し、記事の方向性を確認

### Phase 2: 構成設計

1. **記事タイプの決定**: テーマに最適なタイプを選択し、該当テンプレートを読み込む
2. **アウトライン作成**: テンプレートに沿って見出し構成を作成
3. **ユーザーレビュー**: アウトラインをユーザーに提示して承認を得る

### Phase 3: 下書き執筆

1. **スタイルガイド読み込み**: [style-guide.md](references/style-guide.md) を参照
2. **Markdown 執筆**: Zenn互換形式で記事本文を執筆
3. **ビジュアル要素の配置**: 表、Mermaid図、コードブロック、画像プレースホルダーを挿入

### Phase 4: 公開準備

1. **フロントマター生成**: 下記仕様に従い YAML フロントマターを付与
2. **最終レビュー**: 全体の流れ、誤字、リンク切れを確認
3. **ファイル出力**: `articles/` ディレクトリに `.md` ファイルとして保存

## 記事タイプ選択ガイド

テーマに応じて最適なタイプを選択し、対応するテンプレートを読み込む。

| タイプ | 適したテーマ | テンプレート |
|--------|-------------|-------------|
| **PoC/ハンズオン型** | 新しいサービスやツールを実際に動かして紹介 | [poc-template.md](references/poc-template.md) |
| **概要ガイド/解説型** | サービスの機能を体系的に解説 | [overview-template.md](references/overview-template.md) |
| **比較記事** | 複数の技術/サービスを比較検討 | [comparison-template.md](references/comparison-template.md) |
| **トラブルシューティング** | 問題の発見から解決までの過程を記録 | [troubleshooting-template.md](references/troubleshooting-template.md) |
| **アーキテクチャ解説** | システム構成や設計思想を図解 | [architecture-template.md](references/architecture-template.md) |

## Zenn フロントマター仕様

記事冒頭に以下の YAML フロントマターを付与する。

```yaml
---
title: "記事タイトル"
emoji: "🤖"          # 記事の内容を表す絵文字1つ
type: "tech"          # tech または idea
topics: ["tag1", "tag2", "tag3"]  # 最大5つ、小文字英数字
published: false      # 下書き段階では false
---
```

**生成ルール:**
- `title`: 内容を端的に表す日本語タイトル。疑問形やアクション動詞を含めると読者の関心を引きやすい
- `emoji`: テーマに関連する絵文字を1つ選択
- `type`: 技術記事は `"tech"`、ポエムやアイデア系は `"idea"`
- `topics`: 関連技術のタグ（例: `["azure", "ai", "mcp", "claude"]`）。小文字英数字のみ、最大5つ
- `published`: 下書き時は `false`。ユーザーが公開する際に `true` に変更

---

## 出力形式

- ファイル形式: `.md`（Zenn互換 Markdown）
- 保存先: プロジェクトの `articles/` ディレクトリ
- ファイル名: `{slug}.md`（slug はフロントマターの topics やタイトルから生成、小文字英数字とハイフン）
- 画像パス: `![alt](/images/{slug}/filename.png)` 形式でプレースホルダーを配置
