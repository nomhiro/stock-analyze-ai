# PoC/ハンズオン型テンプレート

新しいサービスやツールを実際に動かしながら紹介する記事向け。セットアップから実行結果まで、読者が再現できる構成。

---

## 構成

```
# はじめに
  - 背景（技術トレンド、動機）
  - 一言要約（引用ブロック）
  - 本記事の目的
  - 参考情報（:::message alert で日付免責 + リンク一覧）
---

# 〇〇とは
  - サービス/技術の概要（2-3段落）
  - 特徴を箇条書きまたは表で整理

# 主な機能
  - 表形式で機能一覧を整理
  - カテゴリごとにグルーピング

| カテゴリ | 機能名 | 説明 |
|---------|--------|------|
| 検索 | search | ... |

---

# セットアップ方法
  ### Step 1: 前提条件
  ### Step 2: インストール
  ### Step 3: 設定
  ### Step 4: 動作確認
  - 各ステップにコマンド例（```bash）
  - 設定ファイル例（```json / ```yaml）
  - 必要に応じてスクリーンショット

---

# ユースケース（2〜4事例）
  ## ユースケース1: 〇〇
    - **課題**: 現状の問題
    - **解決策**: このツールでどう解決するか
    - 実行例（コマンド or スクリーンショット）
    - 結果の説明

  ## ユースケース2: 〇〇
    - 同上の構成

---

# 活用のポイント
  - 実践的なTips を箇条書き（3〜5項目）
  - 各ポイントに具体例を添える

---

# まとめ
  - 本記事で行ったことの振り返り（2〜3文）
  - 発展的な活用への言及

# 参考リンク
  - 公式ドキュメント
  - GitHub リポジトリ
  - 関連記事
```

## ポイント

- セットアップは「コピペで動く」レベルの具体性を目指す
- ユースケースはスクリーンショットで結果を可視化する
- 免責事項（「建設業界の専門家ではないため〜」等）があれば冒頭に記載
