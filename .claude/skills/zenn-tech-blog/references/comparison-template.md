# 比較記事テンプレート

複数の技術、サービス、アプローチを比較検討する記事向け。読者がユースケースに応じて選択できる構成。

---

## 構成

```
# はじめに
  - 比較の動機（なぜこの比較が必要か）
  - 対象読者
  - 比較対象の一覧
  - 参考情報（:::message alert で日付免責 + リンク一覧）
---

# 比較対象の概要
  ## 〇〇とは
    - 2〜3文で簡潔に紹介
    - 公式サイトへのリンク

  ## △△とは
    - 同上

---

# 比較一覧
  - 全体を俯瞰できるマトリックス表

| 観点 | 〇〇 | △△ | □□ |
|------|------|------|------|
| 構築方法 | ... | ... | ... |
| コスト | ... | ... | ... |
| スケーラビリティ | ... | ... | ... |
| 学習コスト | ... | ... | ... |

---

# 詳細比較
  ## 観点1: 構築方法
    - 各対象の詳細説明
    - コード例やスクリーンショットで具体的に示す

  ## 観点2: コスト
    - 料金体系の違い
    - 表形式で整理

  （必要な観点分繰り返す）

---

# ユースケース別おすすめ

| ユースケース | おすすめ | 理由 |
|-------------|---------|------|
| シンプルなQ&A | 〇〇 | ... |
| 複雑なワークフロー | △△ | ... |

---

# まとめ
  - 比較結果の要約
  - 「銀の弾丸はない」旨の注記（ユースケース次第）

# 参考リンク
  - 各対象の公式ドキュメント
  - 料金ページ
```

## ポイント

- 比較表は記事のハイライト。列数が多くなりすぎないよう観点を厳選する
- 主観的な評価には根拠を添える
- 「どちらが良い」ではなく「こういう場合は〇〇」という提示の仕方
- コスト比較は時期によって変わるため、日付免責を必ず記載
