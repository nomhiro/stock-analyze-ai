# トラブルシューティング型テンプレート

問題の発見から解決までの過程を記録する記事向け。同じ問題に遭遇した読者が解決策にたどり着ける構成。

---

## 構成

```
# はじめに
  - 発生した問題の概要（1〜2文）
  - 一言要約（引用ブロック）
  - 本記事の対象読者
  - 参考情報（:::message alert で日付免責 + リンク一覧）
---

# 環境情報

| 項目 | バージョン/値 |
|------|-------------|
| OS | ... |
| ランタイム | ... |
| SDK/ライブラリ | ... |
| サービス | ... |

---

# 問題の症状
  - 何が起きたか（エラーメッセージをコードブロックで表示）
  - スクリーンショットがあれば添付
  - 再現手順

```bash
# エラーが発生するコマンド
$ some-command --flag
Error: something went wrong
```

---

# 原因調査の過程
  ## 仮説1: 〇〇が原因？
    - 調査内容
    - 結果: ❌ これは原因ではなかった

  ## 仮説2: △△が原因？
    - 調査内容
    - 結果: ✅ これが原因だった

---

# 解決策
  - 解決手順をステップバイステップで記載
  - コマンド例やコード修正を具体的に示す
  - 修正前/修正後を並べて差分を明確にする

修正前:
```python
# 問題のあるコード
```

修正後:
```python
# 修正後のコード
```

---

# 再発防止/教訓
  - なぜこの問題が発生したか（根本原因）
  - 今後同様の問題を防ぐための対策
  - 箇条書きで3〜5項目

---

# まとめ
  - 問題と解決策の簡潔な振り返り

# 参考リンク
  - 関連する Issue / Stack Overflow
  - 公式ドキュメントの該当箇所
```

## ポイント

- 環境情報は再現性のために必須
- 調査の試行錯誤も記録する（失敗した仮説も価値がある）
- エラーメッセージは正確にコピペする（検索でヒットするように）
- 修正前/修正後の差分を明示する
