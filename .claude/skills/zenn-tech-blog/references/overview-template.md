# 概要ガイド/解説型テンプレート

サービスやプラットフォームの機能を体系的に解説する記事向け。公式ドキュメントをベースに、全体像を把握できる構成。

---

## 構成

```
# はじめに
  - 背景（なぜこの技術/サービスが重要か）
  - 対象読者
  - 本記事の目的
  - 参考情報（:::message alert で日付免責 + リンク一覧）
---

# 基本概念
  - コアとなる概念を図や数式で説明
  - 例: 「エージェント = プロンプト × ツール」
  - Mermaid 図で関係性を可視化

---

# 全体像
  - 機能の全体マップ（表 or 箇条書き）
  - 各機能の構築順序や依存関係を示す

---

# 機能詳細
  ## 🏷️ 機能1: 〇〇
    - 概要（2〜3文）
    - 主要な設定項目を表形式で整理
    - 実装上の注意点
    - スクリーンショット

  ## 🏷️ 機能2: 〇〇
    - 同上の構成

  （必要な数だけ繰り返す）

---

# 実装上の注意点
  - 実際に触ってみて気づいたポイント
  - 公式ドキュメントに書かれていない tips

---

# まとめ
  - 全体の振り返り
  - 次回予告や発展的な内容への言及

# 参考リンク
  - 公式ドキュメント
  - GitHub リポジトリ
  - 関連記事
```

## ポイント

- 機能ごとのセクションは独立して読めるよう構成する
- 見出しに絵文字を使い、視認性を高める
- Mermaid 図で全体像やアーキテクチャを示す
- 各機能で「概要 → 設定/構成 → 注意点」の流れを統一する
